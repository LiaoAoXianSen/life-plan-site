# 人生规划 App 化改造方向

## 结论

可以改，而且建议改。

但不建议把当前网页内容原样塞进手机壳里，而是应该像 `D:/project/wheel-app` 现在这样，先明确“首页只服务一个核心动作”，再把其余能力拆到二级面板、抽屉或独立页。

## 为什么现在不像 App

当前 `D:/project/life-plan-site` 更像一套功能很多的管理台：

- 首页信息密度太高
- 一个页面承担了太多模块
- 重点动作不够突出
- 手机端会天然显得很长、很碎

## 改造原则

### 1. 首页只保留一个主舞台

人生规划的首页不应该同时展示所有模块。

建议首页只保留：

- 今日视图
- 当前最重要目标
- 一个主操作入口

比如首页可以只围绕“今天该做什么”展开。

### 2. 模块化下沉

把这些内容从首页下沉：

- 历史记录
- 模板
- 资料库
- 统计
- 同步设置

它们不消失，但不再挤在首屏。

### 3. 强化卡片式交互

参考大转盘现在的做法，人生规划也更适合：

- 主卡片
- 底部弹层
- 右上角菜单
- 二级列表页

而不是长网页的连续模块。

## 推荐信息架构

### 首页

- 今日聚焦卡
- 当前目标卡
- 快速新增入口
- 底部主导航

### 清单页

- 待办
- 习惯
- 阶段目标

### 记录页

- 打卡历史
- 复盘记录
- 时间线

### 资源页

- 模板
- 资料
- 预设轮盘或辅助工具

### 我的 / 设置页

- 云同步
- 数据备份
- 展示偏好
- 关于

## 与大转盘的一致风格

为了形成同一家产品的感觉，建议统一：

- 更聚焦的首屏
- 更强的主视觉中心
- 更多使用弹层而不是长页
- 设置和低频功能藏到右上角菜单或底部弹层

## 推荐实施顺序

1. 先只重构信息架构和手机端首页，不急着重写所有功能
2. 再把设置、历史、模板等内容拆到二级区域
3. 最后再把同步能力接到新的统一底座

## 和远程存储的关系

这两个事情可以并行，但最好解耦：

- UI App 化，解决“像不像 App”
- 远程同步底座，解决“多端数据一致性”

建议先让架构先行，边改 UI 边接底座，但不要把两个目标写死在同一份大重构里。
