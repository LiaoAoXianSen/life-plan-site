# 灵感、素材库与全局搜索说明

本文档记录 `life-plan-site` 中灵感状态化、素材库和全局搜索的功能设计，便于后续维护。

## 模块概览

- 灵感仍然属于 `records` 记录体系，类型为 `灵感碎片`，会继续进入时间轴、所有记录和记录详情。
- 灵感池是一个筛选与推进视图，不复制灵感数据，只读取 `records` 中的 `灵感碎片`。
- 素材库存储在独立集合 `materials` 中，用于保存励志金句、AI 提示词、摘抄、观点和方法。
- 全局搜索会跨模块汇总结果，并按模块分组展示。

## 灵感数据结构

`灵感碎片` 记录在普通 record 字段外，额外支持：

- `ideaStatus`：灵感状态，取值为 `待整理`、`待实践`、`实践中`、`已验证`、`已放弃`。
- `ideaTags`：字符串数组，用于灵感标签。
- `ideaNextAction`：下一步动作。
- `ideaTodoId`：关联待办 ID，可为空。
- `ideaConclusion`：实践后的结果结论。

默认规则：

- 新灵感默认状态为 `待整理`。
- `待整理` 和 `待实践` 会被视为未处理灵感。
- `实践中` 或 `已验证` 且没有 `ideaConclusion` 时，会被视为已实践但未写结论。
- 灵感不脱离记录体系，因此仍会出现在时间轴、所有记录、记录详情和全局搜索中。

## 灵感交互

- 记录弹窗中，当记录类型为 `灵感碎片` 时，会显示灵感专属字段。
- 所有记录页支持按灵感状态、未处理、已实践未写结论和标签筛选。
- 灵感池页面支持关键词、状态和标签筛选，并显示状态统计。
- 灵感池中的卡片可直接打开详情或进入编辑推进。

## 素材库数据结构

素材库存储在 `data.materials` 中，每条素材包含：

- `id`：素材 ID。
- `type`：素材类型，取值为 `金句`、`提示词`、`摘抄`、`观点`、`方法`。
- `content`：素材正文。
- `tags`：字符串数组。
- `source`：来源。
- `note`：备注。
- `createdAt`：创建时间。
- `updatedAt`：更新时间。

不维护复习次数，也不维护上次复习时间。

## 素材库随机展示规则

- 用户在素材库展示区勾选若干标签。
- 系统从所有命中这些标签的素材合集里去重。
- 每次随机展示总共 3 条素材。
- 点击“换一批”会重新随机。
- 如果没有勾选标签，则从全部素材中随机展示。

## 搜索范围与规则

全局搜索覆盖：

- 记录：标题、正文、类型、日期、灵感标签、灵感状态、下一步和结论。
- 待办：任务名称、分组、状态、计划时间、子任务和执行备注。
- 目标：目标名称、周期、状态、目标描述和进度。
- 素材库：内容、类型、标签、来源和备注。
- 模板：模板名称、类型、内容和模板待办。
- 转盘公共项：公共项名称、备注、标签和启用状态。

搜索支持：

- 全局搜索：在全部模块中搜索。
- 分组搜索：只搜索指定模块。
- 结果按模块分组，点击结果打开对应详情或跳转到相关页面。

## 页面入口

- 侧边栏新增 `灵感池`、`素材库` 和 `全局搜索`。
- 首页的时间轴仍然展示 `灵感碎片`。
- 所有记录页仍然可以查看和筛选灵感。
- 素材库提供新增、编辑、删除、筛选和随机展示。

## 维护注意

- 新集合 `materials` 已纳入数据归一化、保存、导出、快照和云同步合并。
- 后续如果增加新的素材类型，应同步更新 `MATERIAL_TYPES`、素材弹窗类型选项和本文档。
- 后续如果增加新的灵感状态，应同步更新 `IDEA_STATUS_OPTIONS`、筛选器、状态样式和本文档。
